## 3. 收集的数据及其使用方式

### 3.1 仅存储在您设备上的数据（从不传输）

以下数据仅存储在您的设备上，从不传输到开发者运营的任何服务器：

| 数据 | 目的 | 存储方式 |
|---|---|---|
| BIP39 助记词（加密） | 钱包恢复 | Android Keystore (AES-256-GCM) |
| 私钥（派生，加密） | 交易签名 | Android Keystore (AES-256-GCM) |
| 钱包地址 (RVN) | 显示和交易 | 本地加密存储 |
| 管理员/操作员密钥（Brand 版本） | 资产管理 | Android Keystore (AES-256-GCM) |
| 应用程序设置和偏好 | 应用程序配置 | 本地共享偏好 |

**您的助记词和私钥永远不会离开您的设备。**

### 3.2 NFC 标签验证期间传输的数据

扫描 NFC 标签时，应用程序将以下数据发送到后端服务器以执行密码学验证：

| 数据 | 目的 |
|---|---|
| 资产名称（例如 BRAND/PRODUCT#001） | 在区块链上识别资产 |
| 加密 NFC 计数器（参数 e） | SUN MAC 验证 |
| NFC MAC 值（参数 m） | SUN MAC 验证 |
| 您设备的 IP 地址 | 服务器端速率限制和安全日志记录 |

这些数据是验证 NFC 标签真实性所需的最少数据。验证请求除 IP 地址外不包含任何个人身份信息。

**哪个服务器接收这些数据取决于应用程序配置：**

- **默认/RavenTag 运营的后端**：如果应用程序连接到开发者运营的 RavenTag 后端 (raventag.com)，开发者将按本隐私政策所述接收和处理这些数据。
- **品牌运营的后端**：如果应用程序由品牌编译或配置为连接到其自己的服务器，则该品牌的服务器接收这些数据。开发者不以任何方式接收、访问或处理这些数据。品牌是独立数据控制者，其自己的隐私政策适用。

您可以通过查看应用程序设置中显示的服务器 URL 来识别应用程序连接到哪个后端。

**保留期（开发者运营的后端）**：IP 地址和请求日志最多保留 90 天用于安全和速率限制目的，之后自动删除。

**法律依据（GDPR，开发者运营的后端）**：合法利益（GDPR 第 6(1)(f) 条）：安全监控和欺诈预防。

### 3.3 区块链操作期间传输的数据

当您执行钱包操作（查看余额、发送 RVN、发行资产）时，应用程序与 Ravencoin 网络节点通信。此通信可能包括：

| 数据 | 目的 |
|---|---|
| 您的 Ravencoin 钱包地址 | 查询余额和交易历史 |
| 交易数据 | 向网络广播交易 |
| 您设备的 IP 地址 | 网络通信 |

Ravencoin 区块链是一个公共的去中心化网络。广播到网络的所有交易对任何人都是永久公开可见的。请勿将此钱包用于您希望保持私密的交易。

### 3.4 加载资产图像期间传输的数据

加载 IPFS 托管的资产图像时，应用程序连接到公共 IPFS 网关（如 ipfs.io、cloudflare-ipfs.com）。这些第三方服务可能根据其自己的隐私政策记录您的 IP 地址。

### 3.5 摄像头数据

如果您使用摄像头在应用程序内扫描二维码，摄像头数据仅在您的设备上实时处理，从不存储或传输。

### 3.6 NFC 数据

NFC 标签数据在您的设备上本地读取。原始 NFC 数据（UID、NDEF 记录）在设备上处理，只有派生的验证参数（资产、e、m）按第 3.2 节所述传输。

### 3.7 我们不收集的数据

我们明确不收集：

- 您的姓名、电子邮件地址或任何个人身份信息。
- 设备标识符（IMEI、Android ID、广告 ID）。
- 位置数据。
- 使用分析或遥测。
- 崩溃报告（除非您明确提交）。
- 任何用于广告目的的数据。

## 5. 数据安全

存储在您设备上的所有敏感数据（助记词、私钥、API 密钥）均通过 Android Keystore 系统使用 AES-256-GCM 加密，该系统在可用时使用硬件支持的安全性。

应用程序与开发者后端服务器之间的通信使用 HTTPS/TLS 加密。

尽管采取了这些措施，但没有任何电子存储或传输方法是 100% 安全的。您负责维护设备安全和助记词安全。

---

## 6. 数据保留

- **设备上的数据**：存储至您删除钱包或卸载应用程序。
- **服务器端请求日志**：最多保留 90 天，然后自动删除。
- **区块链数据**：广播到 Ravencoin 区块链的所有交易都是永久公开的，开发者或任何第三方均无法删除。
