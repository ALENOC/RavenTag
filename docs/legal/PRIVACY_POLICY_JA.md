## 3. 収集するデータとその使用方法

### 3.1 デバイスにのみ保存されるデータ（送信しない）

以下のデータはデバイスにのみ保存され、開発者が運営するサーバーには送信されません：

| データ | 目的 | 保存場所 |
|---|---|---|
| BIP39 ニーモニックフレーズ（暗号化） | ウォレット回復 | Android Keystore (AES-256-GCM) |
| 秘密鍵（派生、暗号化） | トランザクション署名 | Android Keystore (AES-256-GCM) |
| ウォレットアドレス (RVN) | 表示とトランザクション | ローカル暗号化ストレージ |
| 管理者/オペレーターキー（Brand バージョン） | 資産管理 | Android Keystore (AES-256-GCM) |
| アプリ設定と設定値 | アプリ設定 | ローカル共有設定 |

**ニーモニックフレーズと秘密鍵はデバイスから外に出ることはありません。**

### 3.2 NFC タグ検証時に送信されるデータ

NFC タグをスキャンすると、アプリは暗号検証を行うために以下のデータをバックエンドサーバーに送信します：

| データ | 目的 |
|---|---|
| 資産名（例：BRAND/PRODUCT#001） | ブロックチェーン上の資産の識別 |
| 暗号化 NFC カウンター（パラメータ e） | SUN MAC 検証 |
| NFC MAC 値（パラメータ m） | SUN MAC 検証 |
| デバイスの IP アドレス | サーバーサイドのレート制限とセキュリティログ |

これらのデータは NFC タグの真正性を確認するために必要な最小限のものです。検証リクエストには IP アドレス以外の個人を特定できる情報は含まれません。

**どのサーバーがこれらのデータを受け取るかはアプリの設定によって異なります：**

- **デフォルト/RavenTag 運営のバックエンド**：アプリが開発者運営の RavenTag バックエンド (raventag.com) に接続している場合、開発者はこのプライバシーポリシーに記載された通りにこれらのデータを受け取り処理します。
- **ブランド運営のバックエンド**：アプリがブランドによって自社サーバーに接続するようコンパイルまたは設定されている場合、そのブランドのサーバーがこれらのデータを受け取ります。開発者はこれらのデータを受け取り、アクセスし、処理することは一切ありません。ブランドが独立したデータ管理者となり、自社のプライバシーポリシーが適用されます。

アプリ設定に表示されるサーバー URL を確認することで、アプリがどのバックエンドに接続しているかを確認できます。

**保存期間（開発者運営のバックエンド）**：IP アドレスとリクエストログはセキュリティとレート制限のため最大 90 日間保存され、その後自動的に削除されます。

**法的根拠（GDPR、開発者運営のバックエンド）**：正当な利益 (GDPR 第 6 条(1)(f)) - セキュリティ監視と不正防止。

### 3.3 ブロックチェーン操作時に送信されるデータ

ウォレット操作（残高確認、RVN 送信、資産発行）を行うと、アプリは Ravencoin ネットワークノードと通信します。この通信には以下が含まれる場合があります：

| データ | 目的 |
|---|---|
| Ravencoin ウォレットアドレス | 残高とトランザクション履歴の照会 |
| トランザクションデータ | ネットワークへのトランザクションブロードキャスト |
| デバイスの IP アドレス | ネットワーク通信 |

Ravencoin ブロックチェーンは公開された分散型ネットワークです。ネットワークにブロードキャストされたすべてのトランザクションは誰でも永続的に公開閲覧できます。プライベートにしたいトランザクションにはこのウォレットを使用しないでください。

### 3.4 資産画像読み込み時に送信されるデータ

IPFS でホストされた資産画像を読み込む際、アプリは公開 IPFS ゲートウェイ (ipfs.io、cloudflare-ipfs.com など) に接続します。これらのサードパーティサービスは、独自のプライバシーポリシーに従って IP アドレスをログに記録する場合があります。

### 3.5 カメラデータ

アプリ内で QR コードをスキャンするためにカメラを使用する場合、カメラデータはデバイス上でリアルタイムにのみ処理され、保存または送信されることはありません。

### 3.6 NFC データ

NFC タグデータはデバイス上でローカルに読み取られます。生の NFC データ（UID、NDEF レコード）はデバイス上で処理され、派生した検証パラメータ（資産、e、m）のみが第 3.2 節で説明されている通り送信されます。

### 3.7 収集しないデータ

私たちは明示的に以下を収集しません：

- 氏名、メールアドレス、その他の個人を特定できる情報。
- デバイス識別子（IMEI、Android ID、広告 ID）。
- 位置データ。
- 使用状況分析またはテレメトリ。
- クラッシュレポート（明示的に送信した場合を除く）。
- 広告目的のデータ。

## 5. データセキュリティ

デバイスに保存されるすべての機密データ（ニーモニックフレーズ、秘密鍵、API キー）は、利用可能な場合はハードウェアバックのセキュリティを使用する Android Keystore システムを介して AES-256-GCM で暗号化されます。

アプリと開発者のバックエンドサーバー間の通信は HTTPS/TLS で暗号化されます。

これらの措置にもかかわらず、電子ストレージや送信方法は 100% 安全ではありません。デバイスとニーモニックフレーズのセキュリティ維持はあなたの責任です。

## 6. データ保存期間

- **デバイス上のデータ**：ウォレットを削除するかアプリをアンインストールするまで保存されます。
- **サーバーサイドリクエストログ**：最大 90 日間保存後、自動的に削除されます。
- **ブロックチェーンデータ**：Ravencoin ブロックチェーンにブロードキャストされたすべてのトランザクションは永続的に公開されており、開発者または第三者が削除することはできません。

## 7. GDPR に基づくあなたの権利

欧州経済領域に居住している場合、個人データに関して以下の権利があります：

- **アクセス権**：私たちが保有するあなたの個人データのコピーを請求する（サーバーサイドログに限定）。
- **訂正権**：不正確なデータの訂正を請求する。
- **削除権**：法的保存義務を条件として、サーバー（サーバーログ）からの個人データの削除を請求する。
- **処理制限権**：データの使用方法の制限を請求する。
- **異議申立権**：正当な利益に基づく処理に異議を申し立てる。
- **データポータビリティ権**：構造化された機械可読形式でデータを受け取る。
- **苦情申し立て権**：イタリアのデータ保護機関（Garante per la protezione dei dati personali、https://www.garanteprivacy.it）に苦情を申し立てる権利があります。

**開発者が運営するデモバックエンド**：開発者のデモバックエンドサーバーは欧州連合内にあります。EU 外からデモアプリインスタンスにアクセスする場合、検証リクエストデータ（第 3.2 節）は GDPR 要件に従って EU に転送され、EU 内で処理されます。

**ブランドが運営するバックエンド**：本番デプロイメントでは、バックエンドサーバーの地理的位置はデプロイしたブランドまたはメーカーによってのみ決定されます。開発者はサードパーティブランドが選択したサーバーの場所を管理または把握していません。適用される国際データ転送ルールは、そのデプロイメントを運営するブランドのルールです。詳細はブランドのプライバシーポリシーをご参照ください。
