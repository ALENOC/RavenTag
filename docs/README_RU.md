**RavenTag: платформа с открытым исходным кодом для борьбы с подделками без доверия третьим сторонам. Она связывает NFC-чипы NTAG 424 DNA с активами блокчейна Ravencoin с использованием протокола RTP-1, позволяя брендам доказывать подлинность физических товаров без зависимости от какого-либо центрального органа.**

## Протокол RTP-1

RTP-1 (RavenTag Protocol v1) определяет JSON-схему, хранящуюся на IPFS и referenced метаданными активов Ravencoin.

### Схема метаданных (v1.1)

```json
{
  "raventag_version": "RTP-1",
  "parent_asset": "FASHIONX",
  "sub_asset": "FASHIONX/BAG001",
  "variant_asset": "SN0001",
  "nfc_pub_id": "<sha256hex>",
  "crypto_type": "ntag424_sun",
  "algo": "aes-128",
  "image": "ipfs://<cid>",
  "description": "<описание продукта>"
}
```

**Описание полей**:
- `raventag_version`: Версия протокола (всегда "RTP-1")
- `parent_asset`: Имя корневого актива (например, "FASHIONX")
- `sub_asset`: Полный путь sub-актива (например, "FASHIONX/BAG001")
- `variant_asset`: Серийный номер уникального токена (например, "SN0001") - присутствует только у уникальных токенов
- `nfc_pub_id`: SHA-256(UID || BRAND_SALT) - публичный отпечаток чипа
- `crypto_type`: Тип NFC-крипто (всегда "ntag424_sun" для NTAG 424 DNA)
- `algo`: Алгоритм шифрования (всегда "aes-128")
- `image`: IPFS-URI изображения продукта (опционально)
- `description`: Описание продукта (опционально)


## Что такое RavenTag?

Контрафакция ежегодно обходится брендам и потребителям в миллиарды. RavenTag решает эту проблему, объединяя три технологии:

**1. Аппаратно защищенный NFC-чип (NTAG 424 DNA)**
Каждый аутентифицированный товар оснащен чипом NXP NTAG 424 DNA. При каждом прикосновении чип генерирует уникальный URL с криптографической подписью через AES-128-CMAC (SUN: Secure Unique NFC). Ключ AES хранится в кремниевой матрице чипа и не может быть извлечен или клонирован.

**2. Блокчейн Ravencoin**
Бренд регистрирует каждый сериализованный товар как уникальный on-chain токен, например `FASHIONX/BAG001#SN0001`. Метаданные токена (хранятся в IPFS) содержат публичный отпечаток чипа: `nfc_pub_id = SHA-256(chip_uid || BRAND_SALT)`.

**3. Суверенное управление ключами бренда**
Ключи AES формируются на сервере бренда из `BRAND_MASTER_KEY`. Ключи никогда не покидают сервер бренда.

## Почему Ravencoin?

[Ravencoin](https://ravencoin.org): форк Bitcoin, созданный специально для выпуска активов:

- **Нативный протокол активов**: корневые активы, суб-активы и уникальные токены являются функциями протокола первого класса. Без смарт-контрактов.
- **Иерархия активов соответствует структуре брендов**: БРЕНД (корневой) / ПРОДУКТ (суб-актив) / СЕРИЙНЫЙ#ТЕГ (уникальный токен).
- **KAWPOW proof-of-work**: алгоритм с сопротивлением ASIC.
- **Без ICO, без предмайнинга**: Ravencoin был запущен честно в январе 2018 года без предварительно выделенных монет.
- **Предсказуемые низкие комиссии**: 500 RVN за корневой актив, 100 RVN за линейку продуктов, 5 RVN за сериализованный товар.

Лицензия RTSL-1.0 требует, чтобы все производные работы использовали исключительно Ravencoin для всех операций с блокчейном.

## Четыре уровня защиты от подделок

| Уровень | Технология | Что нужно злоумышленнику |
|---------|-----------|--------------------------|
| **1. Консенсус Ravencoin** | Владение родительским активом проверяется каждым узлом | Должен владеть родительским активом Ravencoin |
| **2. Привязка nfc_pub_id** | `SHA-256(UID \|\| BRAND_SALT)` в метаданных IPFS | Должен знать `BRAND_SALT` (никогда не публичный) |
| **3. Формирование ключа AES** | `AES-ECB(BRAND_MASTER_KEY, [слот \|\| UID])` на чип | Должен знать `BRAND_MASTER_KEY` |
| **4. Кремний NTAG 424 DNA** | Неклонируемое оборудование NXP | Должен физически клонировать кремний NXP (невозможно) |

## Два Android-приложения

### RavenTag Verify (приложение для потребителей)

Загрузка: [RavenTag-Verify-v1.3.8.apk](https://github.com/ALENOC/RavenTag/releases/latest)

| Возможность | Подробности |
|---|---|
| Сканирование NFC-тегов | Прикосновение к чипу NTAG 424 DNA, результат ПОДЛИННЫЙ / ОТОЗВАННЫЙ |
| Полная верификация | SUN MAC + блокчейн Ravencoin + проверка отзыва |
| Кошелек Ravencoin | BIP44 `m/44'/175'/0'/0/0`, мнемоника BIP39 из 12 слов |
| Многоязычность | EN, IT, FR, DE, ES, ZH, JA, KO, RU |

### RavenTag Brand Manager (приложение для операторов)

Загрузка: [RavenTag-Brand-v1.3.8.apk](https://github.com/ALENOC/RavenTag/releases/latest)

| Возможность | Подробности |
|---|---|
| Выпуск активов Ravencoin | Корневой (500 RVN), суб-актив (100 RVN), уникальный токен (5 RVN) |
| Программирование чипов NTAG 424 DNA | Ключи AES-128 + URL SUN через APDU ISO 7816-4 |
| HD-кошелек | BIP44, локальная подпись UTXO, мнемоника BIP39 из 12 слов |
| Передача / Отзыв / Сжигание | Полное управление жизненным циклом активов |
| Метаданные продукта | Загрузка JSON RTP-1 на IPFS через Pinata, хеш CIDv0 указывается on-chain |

## Роли и уровни доступа

Роли определяются при создании кошелька путем ввода ключа управления (администратора или оператора). Приложение автоматически определяет роль, проверяя ключ на бэкенде. Роль привязана к кошельку и не может быть изменена в настройках.

| Роль | Ключ управления | Права в Android-приложении |
|---|---|---|
| **Администратор** | Ключ администратора (`X-Admin-Key`) | Все операции: выпуск корневых/дочерних активов, выпуск уникальных токенов, отзыв/восстановление, отправка RVN, перевод всех типов активов |
| **Оператор** | Ключ оператора (`X-Operator-Key`) | Только выпуск уникальных токенов. Создание корневых/дочерних активов, отзыв/восстановление, отправка RVN и перевод корневых/дочерних активов заблокированы. |

**Сценарий использования:** Администратор бренда может предварительно настроить несколько устройств операторов на одном кошельке с активами-владельцами. Каждое устройство оператора может выпускать уникальные токены без доступа к управлению активами или кошельку.

## Поведение при прикосновении к NFC-тегу

**Приложение установлено:** URL тега перехватывается непосредственно RavenTag Verify. Приложение открывает экран сканирования и выполняет полную верификацию.

**Приложение не установлено:** URL тега открывается в браузере, отображающем страницу установки с логотипом и ссылкой для загрузки.
